# 羊羊运动打卡系统

## 项目简介

羊羊运动打卡系统是一个基于 Python Flask + 前端分离的网站，用于记录"羊羊"每个月的运动打卡情况。该系统支持状态管理、数据统计和移动端适配，使用 SQLite 存储打卡数据。

## 核心功能

- **日历展示**：显示当月日历，每行一个星期，当天高亮显示
- **状态管理**：每个格子可设置为已运动、未运动、已请假三种状态
- **月份切换**：支持上个月、下个月切换，查看历史打卡记录
- **数据统计**：自动汇总当月已运动、未运动、已请假的次数
- **数据存储**：使用 SQLite 持久化存储每个月、每天的打卡数据
- **响应式设计**：适应手机屏幕，支持移动端操作

## 技术栈

- **后端**：Python 3.8+, Flask 2.0+, SQLite
- **前端**：HTML5, CSS3, JavaScript
- **依赖库**：Flask, Flask-CORS

## 安装和运行

### 1. 安装依赖

```bash
# 进入 backend 目录
cd backend

# 安装依赖
pip install -r requirements.txt
```

### 2. 启动后端服务

```bash
# 在 backend 目录下运行
python app.py

# 服务将运行在 http://127.0.0.1:5000
```

### 3. 访问前端页面

直接打开 `frontend/index.html` 文件即可访问网站。

## 使用方法

1. **查看当月打卡**：打开网站后，默认显示当月的日历和打卡情况
2. **切换月份**：点击顶部的「上个月」或「下个月」按钮切换月份
3. **设置打卡状态**：点击日历中的任意格子，在弹出的菜单中选择状态（已运动、未运动、已请假）
4. **查看统计数据**：页面顶部的统计区域会自动显示当月的打卡统计
5. **移动端操作**：在手机上打开页面，可通过触摸操作进行同样的功能

## 数据说明

- **数据库**：使用 SQLite 数据库，文件名为 `fitness.db`
- **表结构**：`fitness_records` 表存储打卡记录
  - `id`：记录ID
  - `year`：年份
  - `month`：月份
  - `day`：日期
  - `status`：状态（exercised/absent/leave）
  - `created_at`：创建时间
  - `updated_at`：更新时间

## API 接口

### 获取日历数据

- **URL**：`/api/calendar`
- **方法**：GET
- **参数**：
  - `year`：年份
  - `month`：月份
- **响应**：
  ```json
  {
    "days": [
      {"day": 1, "status": "exercised"},
      {"day": 2, "status": "absent"},
      ...
    ],
    "stats": {
      "exercised": 5,
      "absent": 10,
      "leave": 2
    }
  }
  ```

### 更新打卡状态

- **URL**：`/api/status`
- **方法**：POST
- **参数**：
  ```json
  {
    "year": 2026,
    "month": 2,
    "day": 1,
    "status": "exercised"
  }
  ```
- **响应**：
  ```json
  {
    "success": true,
    "message": "状态更新成功"
  }
  ```

  - 统计信息调整为横向排列

## 注意事项

1. 确保后端服务在运行状态，否则前端无法获取和更新数据
2. 首次运行时，系统会自动创建数据库和表结构
3. 建议定期备份 `fitness.db` 文件，防止数据丢失
4. 本项目使用的是开发服务器，生产环境部署时建议使用 WSGI 服务器
