# 羊羊运动打卡系统 - 产品设计文档

## 1. 产品概述

**项目名称**：羊羊运动打卡系统

**项目目标**：创建一个基于 Python Flask + 前端分离的网站，用于记录"羊羊"每个月的运动打卡情况，支持状态管理、数据统计和移动端适配。

**核心价值**：
- 直观的日历视图展示每日运动状态
- 方便的月份切换功能
- 自动统计当月运动数据
- 支持移动端操作
- 数据持久化存储

## 2. 核心功能

### 2.1 功能模块

| 模块 | 功能描述 |
|------|----------|
| 日历展示 | 显示当月日历，每行一个星期，当天高亮 |
| 状态管理 | 每个格子可设置为：已运动、未运动、已请假 |
| 月份切换 | 支持上个月、下个月切换 |
| 数据统计 | 自动汇总当月已运动、未运动、已请假次数 |
| 数据存储 | 使用 SQLite 存储每个月、每天的打卡数据 |
| 响应式设计 | 适应手机屏幕，支持移动端操作 |

### 2.2 页面结构

| 页面 | 功能描述 |
|------|----------|
| 主页 | 显示当月日历、状态管理、月份切换、数据统计 |

## 4. 数据结构

### 4.1 数据库设计

**表名**：fitness_records

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 记录ID |
| year | INTEGER | NOT NULL | 年份 |
| month | INTEGER | NOT NULL | 月份 |
| day | INTEGER | NOT NULL | 日期 |
| status | TEXT | NOT NULL | 状态：exercised/absent/leave |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- CREATE INDEX idx_year_month_day ON fitness_records(year, month, day);

### 4.2 API 设计

| API 路径 | 方法 | 功能描述 | 请求参数 | 响应格式 |
|----------|------|----------|----------|----------|
| /api/calendar | GET | 获取指定月份的日历数据 | year, month | `{"days": [{"day": 1, "status": "exercised"}], "stats": {"exercised": 5, "absent": 10, "leave": 2}}` |
| /api/status | POST | 更新指定日期的状态 | year, month, day, status | `{"success": true, "message": "状态更新成功"}` |

## 6. 交互流程

### 6.1 页面加载流程

1. 访问网站主页
2. 前端获取当前年月
3. 调用 API 获取当月日历数据
4. 渲染日历和统计信息
5. 高亮显示当天

### 6.2 状态更新流程

1. 用户点击日历格子
2. 弹出状态选择菜单（已运动、未运动、已请假）
3. 用户选择状态
4. 前端调用 API 更新状态
5. API 保存数据到 SQLite
6. 前端更新日历格子显示
7. 前端重新计算并更新统计信息

### 6.3 月份切换流程

1. 用户点击「上个月」或「下个月」按钮
2. 前端计算目标年月
3. 调用 API 获取目标月份的日历数据
4. 前端更新日历显示和统计信息
5. 保持当前月份的状态选择

## 7. 移动端适配

### 7.1 设计考虑

- **触摸友好**：按钮和格子大小适合触摸操作（至少 44x44px）

- **性能优化**：减少 DOM 操作，优化渲染速度

- **平板**：iPad, Android 平板
- **浏览器**：Chrome, Safari, Firefox
